# 目的

friend の所属タレントをタレントDB に登録し、案件DB の Select オプションに反映することで、以後の sync で正しく自動マッチングできるようにする。

# 前提コンテキストの取得

1. `notion-search(query="task-manager 設定", page_size=5)` で設定ページの page_id を取得
2. `notion-fetch` で設定ページ → メタ情報DB の `data_source_id` を抽出（`META_DSID`）
3. `notion-search(query="メタ情報DB", data_source_url="collection://"+META_DSID, page_size=5)` で config レコード（通常1件）を特定し、`notion-fetch` で:
   - 案件DB の data_source_id（`DEALS_DSID`）
   - タレントDB の data_source_id（`TALENTS_DSID`）

# 手順

## 1. ユーザーから情報を聞く

対話形式で以下を取得:

1. **名前** (必須): 「タレントの本名 or 主に使う芸名は？」
2. **エイリアス** (任意): 「他に呼ばれる名前があれば、カンマ区切りで。例: 太郎, タロウ, T-ROCK。なければスキップ可」
3. **関連メアド** (任意): 「このタレント宛のメールが届くアドレスや本人のアドレスがあれば、カンマ区切りで。なければスキップ可」

ユーザーがメッセージ内で既に全項目を提示している場合（例: 「山田太郎を追加して、エイリアスはタロウ」）は再質問せずそのまま進む。

## 2. タレントDB に追加

`notion-create-pages` で `TALENTS_DSID` に新規ページ作成:

- 名前: <入力値>
- エイリアス: <カンマ区切り文字列、空ならスキップ>
- 関連メアド: <カンマ区切り文字列、空ならスキップ>

## 3. 案件DB の「タレント名」Select に追加

Notion Connector の DDL には **`ADD OPTION` 構文は存在しない**。代わりに `ALTER COLUMN ... SET SELECT(...)` で **既存オプション + 新規オプション** を含めた全リストで上書きする：

1. まず `notion-fetch(id="collection://<DEALS_DSID>")` で案件DBのスキーマを取得
2. レスポンス内の `"タレント名"` プロパティの `options` 配列から既存オプション名と色を抽出
3. 既存リスト + 新規 `<名前>` を含む完全な SET 文を組み立てる:

```sql
ALTER COLUMN "タレント名" SET SELECT('未設定':gray, '<既存A>':<色A>, ..., '<新規名前>':blue)
```

4. `notion-update-data-source(data_source_id=DEALS_DSID, statements=<上記>)` を実行

新規オプションの色は `blue` 既定。既に同名オプションがある場合は、追加せず手順 4 へ進む。
（バージョンが v1.0-en の場合は英語名 `talent` を使う）

## 4. 完了報告

```
✅ タレントを追加しました

名前: <X>
エイリアス: <Y> （未入力なら "なし"）
関連メアド: <Z> （未入力なら "なし"）

タレントDB: <ページ URL>
案件DB の「タレント名」Select に追加済み

次回の /tm-sync 以降、このタレント宛のメールが自動マッチングされます。
```

# 出力フォーマット

手順 4 の通り。

# 判断指針 / エッジケース

- 同名タレントが既に存在する場合: ユーザーに「同名のタレント <既存名> がいます。別人として追加しますか？それともエイリアス/メアドの追記でよいですか？」と確認。
- 関連メアドの形式チェック: `@` を含むか軽く検証。明らかに不正なら確認。
- **手順 2（タレントDB追加）が失敗した場合**: 手順 3（Select オプション追加）は **実行しない**。エラー内容（Notion 側のメッセージ）をユーザーに提示して終了する。タレントDB に未登録のものを Select だけに追加すると整合性が崩れるため。
- **手順 3（Select オプション追加）が失敗した場合**: 手順 2 のタレントDB追加はロールバックしない（部分成功として残す）。ユーザーに「タレントDB には追加できましたが、案件DB の Select オプション追加に失敗しました。Notion 案件DB で「タレント名」プロパティに `<名前>` を手動追加してください」と案内。

# 完了条件

- タレントDB に新規行が作成されている
- 案件DB の「タレント名」Select にオプションが追加されている
- ユーザーに完了通知（タレントDB URL 含む）が表示されている

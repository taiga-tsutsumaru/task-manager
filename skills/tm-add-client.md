# 目的

新規取引先・既存取引先をクライアントDB に登録し、案件DB の Select オプションに反映する。送信元ドメインを正しく登録しておくことで、以後の sync で送信元メアドから自動マッチングできるようになる。

# 前提コンテキストの取得

1. `notion-search(query="task-manager 設定", page_size=5)` で設定ページの page_id を取得
2. `notion-fetch` で設定ページ → メタ情報DB の `data_source_id` を抽出（`META_DSID`）
3. `notion-search(query="メタ情報DB", data_source_url="collection://"+META_DSID, page_size=5)` で config レコード（通常1件）を特定し、`notion-fetch` で:
   - 案件DB の data_source_id（`DEALS_DSID`）
   - クライアントDB の data_source_id（`CLIENTS_DSID`）

# 手順

## 1. ユーザーから情報を聞く

対話形式で以下を取得:

1. **会社名** (必須): 「クライアントの正式名称は？」
2. **ドメイン** (任意だが推奨): 「メールアドレスのドメイン部分（@より後ろ）を、カンマ区切りで。例: example.co.jp, example.com。複数の子会社・部署で違うドメインを使うなら全部入れてOK。なければスキップ可」
3. **エイリアス** (任意): 「他の呼び方や略称があれば、カンマ区切りで。例: A社, ABC, ABCコーポレーション。なければスキップ可」

ユーザーがメッセージ内で既に全項目を提示している場合は再質問せずそのまま進む。

## 2. クライアントDB に追加

`notion-create-pages` で `CLIENTS_DSID` に新規ページ作成:

- 会社名: <入力値>
- ドメイン: <カンマ区切り文字列、空ならスキップ>
- エイリアス: <カンマ区切り文字列、空ならスキップ>

## 3. 案件DB の「クライアント名」Select に追加

Notion Connector の DDL には **`ADD OPTION` 構文は存在しない**。代わりに `ALTER COLUMN ... SET SELECT(...)` で **既存オプション + 新規オプション** を含めた全リストで上書きする：

1. `notion-fetch(id="collection://<DEALS_DSID>")` で案件DBのスキーマを取得し、「クライアント名」プロパティの既存 `options` を読む
2. 既存リスト + 新規 `<会社名>` を含む完全な SET 文を組み立てる:

```sql
ALTER COLUMN "クライアント名" SET SELECT('未設定':gray, '<既存A>':<色A>, ..., '<新規会社名>':green)
```

3. `notion-update-data-source(data_source_id=DEALS_DSID, statements=<上記>)` を実行

新規オプションの色は `green` 既定。既に同名オプションがあれば追加せず手順 4 へ。
（バージョンが v1.0-en の場合は英語名 `client` を使う）

## 4. 完了報告

```
✅ クライアントを追加しました

会社名: <X>
ドメイン: <Y> （未入力なら "なし"）
エイリアス: <Z> （未入力なら "なし"）

クライアントDB: <ページ URL>
案件DB の「クライアント名」Select に追加済み

次回の /tm-sync 以降、このドメインから来たメールが自動マッチングされます。
```

ドメインが未入力なら追加で警告:

> ⚠️ ドメインが未登録のため、メール送信元による自動マッチングは効きません。後から /tm-add-client で同じ会社名で上書き登録するか、Notion で直接編集してください。

# 出力フォーマット

手順 4 の通り。

# 判断指針 / エッジケース

- 同名会社が既に存在する場合: 「同名 <既存名> が登録済みです。ドメイン/エイリアスを追記する更新を行いますか？それとも別エントリとして追加しますか？」と確認。
- ドメイン形式: `@` を含まないこと、ドットを含むこと、を軽く確認。`@example.com` のように `@` 付きで来たら `@` を除去して保存。
- **手順 2（クライアントDB追加）が失敗した場合**: 手順 3（Select オプション追加）は **実行しない**。エラー内容（Notion 側のメッセージ）をユーザーに提示して終了する。クライアントDB に未登録のものを Select だけに追加すると整合性が崩れるため。
- **手順 3（Select オプション追加）が失敗した場合**: 手順 2 のクライアントDB追加はロールバックしない（部分成功として残す）。ユーザーに「クライアントDB には追加できましたが、案件DB の Select オプション追加に失敗しました。Notion 案件DB で「クライアント名」プロパティに `<会社名>` を手動追加してください」と案内。

# 完了条件

- クライアントDB に新規行が作成されている
- 案件DB の「クライアント名」Select にオプションが追加されている
- ユーザーに完了通知（クライアントDB URL 含む）が表示されている

# 目的

案件DB から **自社で動くべき案件**（ボール所在=🔴 自社対応中 もしくは 次回アクションの期日が今日〜1日後）を抽出し、放置リスクが高い順に一覧化する。各案件の概要と次回アクションを AI が整形し、すぐ動けるリンク付きで提示する。

# 前提コンテキストの取得

1. `notion-search` で `task-manager 設定` を取得
2. 設定ページ配下のメタ情報DBから config レコードを `notion-fetch`:
   - 案件DB の data_source_id（`DEALS_DSID`）
   - バージョン（プロパティ名のロケール判定用）

# 手順

## 1. 案件DB から要対応案件を抽出（一括取得 → メモリでフィルタ）

Notion Connector の `notion-search` は意味検索ベースで、Boolean プロパティの完全一致や `NOT IN` 等の構造化フィルタはサポートされない / 信頼性が低い。`notion-fetch` は ID 単位取得のみ。そのため次の手順を取る:

1. `notion-search` で `data_source_url` を `DEALS_DSID` に絞り、`query` は空文字（または `*`）で **案件DBを全件取得**。`nextCursor` を辿ってページネーション対応。
2. 取得した全案件を **このSkill実行中の Claude 自身がメモリ上でフィルタリング**:
   - `ステータス NOT IN ('完了', '中断')` （必須）
   - かつ以下のいずれか:
     - `ボール所在 = '🔴 自社対応中'`、または
     - `こちらの返信要否 = true`（旧スキーマ互換）、または
     - `次回アクションの期日 <= 今日+1日`（期日が直近のもの）
3. 該当ゼロなら「✅ 現在、対応が必要な案件はありません。」とだけ返して終了。

> パフォーマンス注記: この一括取得は案件数が増えると重い。MVP の想定は数十〜数百件のため許容範囲。100 件を超え始めたら出力末尾に「※ 案件数が多くなってきました。Notion 上で View（フィルタ済みビュー）を作成して直接見る運用への切り替えを検討してください」と添える。

## 2. ソート

優先順位:
1. **次回アクションの期日** 昇順（期日が今日以前のものが最上位）
2. 期日が同じ・空なら **最終やり取り日** 昇順（古いほうが上）
3. 最終やり取り日も空の案件は最後尾

## 3. 各案件の要点を生成

それぞれについて以下を取得・整形:

- 案件名（先頭が `[要編集]` ならそれも残す）
- **案件概要**（不変サマリ、AI 生成）
- ステータス、ボール所在
- **次回アクションとその期日**（「あと N 日」「期日超過」表示）
- 納期と「あと○日」（今日からの差分。納期超過なら `❗超過 -N日`）
- 最終やり取り日と「N日前」
- AIメモから直近の追記要点 1 行
- Notion ページ URL、最新メールへのリンク

## 4. 出力

```
🚨 要対応案件: <N>件

1. <案件名>
   📋 概要: <案件概要>
   ステータス: <X> / ボール: <ボール所在>
   ⏰ 次回アクション: <内容> (期日 YYYY-MM-DD / あと N 日 / 期日超過 -N 日)
   納期: YYYY-MM-DD (あと N 日)
   直近の動き: <AIメモ最新 1 行> (最終やり取り: N日前)
   → Notion: <link>
   → 最新メール: <link>

2. ...

## サマリ
- 期日超過の次回アクション: <N>件
- 納期超過: <N>件
- 今週中納期: <N>件
- 14日以上やり取りなし: <N>件
```

`次回アクションの期日` が今日以前なら要点行頭に `🔥`。
最終やり取り日が 14 日以上前なら要点行頭に `⚠️`。
納期超過の案件は案件名行頭に `❗`。

# 出力フォーマット

手順 4 の通り。要点は冗長な転載でなく、メール内容のエッセンス（何を聞かれているか / こちらが何をすべきか）を 1〜2 行で。

# 判断指針 / エッジケース

- 最終やり取り日が空 → ソートは最後尾。「最終やり取り: 不明」と表示。
- 重要メモに `[AI要確認 ...]` がある案件は要点行頭に `🔶` を付け、AI が何を迷ったかを 1 行で添える。
- 納期が空の場合は「納期: 未設定」と表示。
- 案件数が 20 件を超える場合、上位 20 件を表示し末尾に「他 X 件は省略。全件は Notion 案件DB を直接参照してください。<link>」と添える。

# 完了条件

- 要対応案件が放置リスク順に表示されている
- 各案件に Notion / Gmail のリンクが付いている
- サマリ（超過・今週中・14日放置の件数）が末尾にある

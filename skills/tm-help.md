# 目的

friend が「あれ、何ができるんだっけ」となったときに、全 Skill の概要を一覧表示する。各 Skill の呼び方と一言説明、自然言語での呼び出し例を提示する。

# 前提コンテキストの取得

特になし（このSkillは静的なヘルプを返すだけ）。ただし可能であれば `notion-search` で `task-manager 設定` の存在を確認し、未セットアップなら最初に `/tm-setup` を案内する。

# 手順

## 1. セットアップ状態の確認（任意）

`notion-search("task-manager 設定")` で設定ページが見つかるか確認。見つからなければヘルプ冒頭に:

> ⚠️ まだセットアップが終わっていないようです。最初に `/tm-setup` を実行してください。

を添える。

## 2. ヘルプ本文を表示

```
🛠️ task-manager Skill 一覧

## 毎日使う
- /tm-pending : 今すぐ返信すべき案件一覧
  例: 「ペンディング案件見せて」「今日対応するやつ」
- /tm-sync : Gmail を取り込んで案件DBを更新
  例: 「メール取り込んで」「sync して」「最近のメール案件化」

## 週次
- /tm-report : 今週の動き・入金見込みサマリ
  例: 「週次レポート」「今週のまとめ出して」

## 登録
- /tm-add-talent : タレントを追加
  例: 「山田太郎を追加」「タレント登録」
- /tm-add-client : クライアントを追加
  例: 「A社を取引先に追加」「クライアント登録」

## 初期化（通常 1 回のみ）
- /tm-setup : Notion 上に必要な DB を一括作成
  例: 「セットアップして」「初期化」

## ヘルプ
- /tm-help : このメッセージを表示

---

## Skill を使わずに自然言語で頼んでもOK

個別の更新・問い合わせ・メール下書き作成は Skill を経由しなくても Claude に直接頼めます:

- 「A社の○○案件、ステータスを納品済に変えて、納期も来週に伸ばして」
- 「先週新しく入った案件まとめて」
- 「Bさんに送る△△の件、返信下書き作って」
- 「先月の入金待ち案件、まだ振り込まれてないやつ教えて」

Claude が Notion / Gmail Connector を直接呼び出して対応します。

## 困ったら
- スキーマや設定を確認したい → Notion の「task-manager 設定」ページを開く
- うまく動かない → 「<操作> がエラーになった」と Claude に伝えてください
```

# 出力フォーマット

手順 2 の通り。

# 判断指針 / エッジケース

- 未セットアップ時は冒頭に警告を入れて `/tm-setup` を促す。
- 設定ページが見つからないエラーが Connector の問題（権限不足など）の可能性もあるため、「Notion Connector の接続状態も確認してください」を併記。

# 完了条件

- 全 Skill が呼び方と例とともに一覧化されている
- 自然言語での頼み方の例が提示されている
